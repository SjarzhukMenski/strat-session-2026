# Подготовка к стратсессии 2026

Веб-проект для подготовки к стратегической сессии компании на 2026 год, направленный на сбор информации о процессах автоматизации и требований к управленческим дашбордам.

## Описание проекта

Проект представляет собой систему из трёх взаимосвязанных веб-страниц, предназначенных для сбора структурированной информации от сотрудников компании перед стратегической сессией 2026 года.

### Основные цели проекта

1. **Цифровизация и автоматизация** - выявление рутинных операций для последующей автоматизации
2. **Data-driven управление** - переход от управления "по ощущениям" к управлению "по данным"
3. **Структурированный сбор требований** - формализация потребностей в управленческих инструментах

## Структура проекта

### 1. board.html - Главная страница

Стартовая страница-меню с двумя направлениями работы:

- **Описание рутинных операций для автоматизации** - переход к форме фиксации проблем и узких мест
- **Управленческие дашборды** - переход к форме по ключевым метрикам и отчётам для ТОП-уровня

**Особенности:**
- Адаптивный дизайн с поддержкой мобильных устройств
- Вводный текст о важности цифровизации в 2026 году
- Минималистичный интерфейс с карточками-ссылками

### 2. index.html - Форма "Описание рутинных операций для автоматизации"

Страница с встроенной Google-формой для сбора информации о повседневных рутинных операциях сотрудников.

**Функционал:**
- Встроенная Google-форма с 11 вопросами
- Подробная инструкция по заполнению в правой колонке
- AI-чат помощник на базе n8n для помощи в заполнении
- Адаптивная компоновка (desktop: форма слева + инструкция справа; mobile: выдвижная панель снизу)

**Собираемая информация:**
1. ФИО сотрудника
2. Подразделение компании
3. Название операции (5-7 слов)
4. Пошаговое описание операции
5. Категория операции (ввод данных, проверка, отчёты и др.)
6. Частота выполнения
7. Время выполнения (в минутах)
8. Количество человек, выполняющих операцию
9. Главная проблема или "боль" операции
10. Последствия ошибки
11. Используемые системы и инструменты (1С, WMS, Office и др.)

**Категории операций:**
- Ввод и перенос данных
- Проверка и сверка информации
- Формирование отчетов
- Создание и оформление документов
- Поиск и сбор информации
- Согласование и утверждение
- Коммуникация и уведомления
- Планирование и распределение
- Обработка транзакций и системных операций

### 3. dash.html - Форма "Управленческие дашборды 2026"

Страница с Google-формой для сбора требований к управленческим дашбордам от руководителей.

**Функционал:**
- Встроенная Google-форма для описания показателей дашбордов
- Детальная инструкция с FAQ в формате аккордеонов
- AI-чат помощник, специализированный на дашбордах
- Адаптивный интерфейс с выдвижной панелью на мобильных устройствах

**Типы дашбордов:**
1. **Ежемесячный (Результирующий)** - итоговые цифры за месяц
2. **Еженедельный (Оперативный)** - "пульс" подразделения для корректировки действий
3. **Аналитический (Исследовательский)** - поиск причин и закономерностей
4. **Ежедневный (Мониторинг)** - критические отклонения, требующие быстрой реакции

**Направления деятельности:**
- Общее управление компанией
- Финансы и учет
- Коммерция
- Логистика

**Для каждого показателя собирается:**
1. Название показателя
2. Управленческий смысл (какие решения принимаются)
3. Формула или логика расчёта
4. Необходимые данные для расчёта
5. Источники данных (системы)

## Технические особенности

### Используемые технологии

- **HTML5** - семантическая разметка
- **CSS3** - адаптивный дизайн с Flexbox
- **JavaScript (ES6+)** - интерактивность и логика UI
- **n8n Chat Widget** - встроенный AI-помощник
- **Google Forms** - сбор данных

### Адаптивность

Проект использует mobile-first подход:
- **Desktop (> 900px)**: две колонки (форма + инструкция)
- **Mobile (≤ 900px)**: выдвижная нижняя панель (bottom sheet) с инструкциями

### Интеграции

**AI-чат помощник (n8n):**
- index.html: webhook для помощи по заполнению формы рутинных операций
- dash.html: webhook для помощи по формированию требований к дашбордам
- Streaming-режим для быстрых ответов
- Кастомизированная локализация на русском языке

### Стилистика

- Современный минималистичный дизайн
- Цветовая схема: светлые оттенки (#f4f6f9, #f9fafc, white)
- System fonts для нативного вида на разных платформах
- Аккордеоны для сворачиваемых блоков контента
- Hover-эффекты и плавные переходы

## Особенности UX

1. **Защита от случайного закрытия** - `beforeunload` event предупреждает о несохраненных данных
2. **Sticky header** на мобильных - заголовок инструкций всегда виден
3. **Визуальные хэндлы** - полоска-индикатор для интерактивных элементов
4. **Контекстные примеры** - готовые кейсы заполнения форм
5. **FAQ в аккордеонах** - быстрый доступ к частым вопросам

## Регламент использования

### Сроки
- **Дедлайн**: 09.12.2025
- **Время заполнения**:
  - Форма рутинных операций: 10-15 минут
  - Форма дашбордов: 10-15 минут на каждый тип дашборда

### Требования к заполнению

**Рутинные операции:**
- Все 11 вопросов обязательные
- Минимум 3-5 шагов в описании операции
- Фокус на проблемах, а не решениях

**Дашборды:**
- Минимум 2 прохода (ежемесячный + еженедельный)
- Минимум 3 показателя на дашборд
- Описание управленческого смысла каждого показателя

## Запуск проекта

Проект работает как статические HTML-страницы и не требует серверной части:

1. Откройте `board.html` в браузере
2. Выберите нужное направление (рутинные операции или дашборды)
3. Заполните форму, используя инструкции и AI-помощник
4. Отправьте ответы
